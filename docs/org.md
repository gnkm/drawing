# drawing — 仕様草稿

## 概要

**drawing** は、箱にくじを入れてランダムに引き出すシングルページアプリケーション（SPA）です。追加したくじを引き、引き出された順番を画面に表示します。

## 目的

- くじの追加・抽選・結果表示を、1 画面で完結させる
- 抽選結果の順序を記録し、ユーザーが確認できるようにする

## 用語

| 用語 | 説明 |
|------|------|
| 箱 | 未抽選のくじが入るコンテナ |
| くじ | ユーザーがフォームから追加する抽選対象の要素 |
| 抽選結果 | 箱から引き出されたくじの履歴（引き出された順番） |

## 画面構成

1 画面に以下の UI を配置する。

### レイアウト

```text
┌─────────────────────────────┬──────────┐
│ 箱（上）                     │          │
│                             │  フォーム │
│                             │  （右）   │
├─────────────────────────────┤          │
│ 抽選結果（下）               │          │
└─────────────────────────────┴──────────┘
```

- 左側: 上に箱、下に抽選結果
- 右側: くじ追加フォーム、抽選操作、リセット操作

ルートパスは `/` とする。drawing 本体をトップページに配置する。

| 領域 | 内容 |
|------|------|
| 箱 | 未抽選のくじをカードとして表示するエリア |
| くじ追加フォーム | くじの内容を入力する単一テキストフィールドと「くじを追加」ボタン |
| 抽選操作 | 「くじを引く」ボタン（箱が空のときは disabled） |
| リセット操作 | 「クリア」ボタン（箱も履歴も空のときは disabled） |
| 抽選結果 | 引き出されたくじを表示するエリア（新しいものが上） |

### 空状態

箱および抽選結果が空のとき、それぞれプレースホルダーまたは説明文を表示する。

## デザイン

[Web Interface Guidelines](https://github.com/vercel-labs/web-interface-guidelines) に従う。

## 機能要件

### FR-1: くじの追加

- ユーザーは単一テキストフィールドに内容を入力し、「くじを追加」ボタンで箱にくじを追加できる
- 追加されたくじには、`hsl(ランダムな色相, 70%, 60%)` 形式の色が自動的に割り当てられる
  - 色相は 0〜360 の整数をランダム生成する
  - 色相の重複は許容する
- 追加されたくじは箱の中にカードとして表示される
- くじを追加した後、入力フィールドはクリアする
- 同じ内容のくじを複数追加できる

#### バリデーション

| 条件 | 挙動 |
|------|------|
| 空文字 | 追加不可 |
| 空白のみ | 追加不可 |
| 最大文字数 | 20 文字 |
| 入力が空 | 「くじを追加」ボタンを disabled にする |

### FR-2: くじの抽選

- ユーザーは「くじを引く」ボタンで、箱の中からランダムに 1 つくじを引き出せる
- 残りくじから均等確率（uniform random）で抽選する
- 箱が空のときは「くじを引く」ボタンを disabled にする
- 引き出されたくじは箱から取り除かれる
- 引き出された順番は保存される
- 抽選時のアニメーションや演出は不要（即座に結果を反映する）

### FR-3: 抽選結果の表示

- 引き出されたくじを表示する（新しいものが上）
- 各結果に順位番号を表示する（1 番目、2 番目、…。番号は引き出し順）
- 表示内容には、くじの内容および割り当てられた色を含める
- 抽選結果は永続化しない（ページリロード後は保持されない）

### FR-4: クリア

- ユーザーは「クリア」ボタンで、箱と抽選結果を初期状態に戻せる
- 箱に入っている未抽選のくじをすべて削除する
- 抽選結果の履歴をすべて削除する
- フォームの入力内容もリセットする
- 確認ダイアログは表示しない
- 箱も履歴も空のときは「クリア」ボタンを disabled にする

## ユーザー操作フロー

```text
1. フォームにくじの内容を入力する（1〜20 文字）
2. 「くじを追加」を押す → 箱にくじがカードとして追加される（色はランダム）→ 入力フィールドがクリアされる
3. 2 を必要な数だけ繰り返す
4. 「くじを引く」を押す → 箱からランダムに 1 つ引き出される
5. 引き出されたくじが抽選結果エリアの先頭（上）に表示される
6. 4 を箱が空になるまで繰り返す
7. （任意）「クリア」を押す → 箱・抽選結果・フォーム入力が初期状態に戻る
```

## データ

### くじ（未抽選）

| 項目 | 説明 |
|------|------|
| 内容 | 単一テキストフィールドから入力された文字列（1〜20 文字） |
| 色 | 追加時に `hsl(0〜360 の整数, 70%, 60%)` で生成した識別用の色 |

### 抽選結果

| 項目 | 説明 |
|------|------|
| 順番 | 引き出された順序（1 番目、2 番目、…） |
| くじ | 引き出されたくじの内容と色 |

箱の内容および抽選結果は、いずれもアプリ実行中のメモリ上のみ保持する。localStorage 等への保存は行わない。

## 技術スタック

[react-template](https://github.com/gnkm/react-template) をベースとする。
必要に応じて取捨選択する。

実装開始時に、既存のデモページ（counter, form など）は削除する。

| カテゴリ | ライブラリ |
|---------|-----------|
| UI | React 19、Tailwind CSS 4 |
| ルーティング | TanStack Router |
| フォーム | controlled input（Zod は使わない） |
| 状態管理 | Zustand |
| ビルド | Vite 8 + React Compiler |
| Lint / Format | Biome |
| テスト | Vitest + Testing Library |
